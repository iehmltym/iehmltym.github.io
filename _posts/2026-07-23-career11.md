# 第一章：互联网到底是怎么工作的（最重要）

先忘掉所有缩写。

假设：

你打开浏览器，输入：

```text
www.google.com
```

然后按 Enter。

到底发生了什么？

---

## 第一步：不知道 Google 在哪里

你的电脑只认识：

> **IP 地址**

例如：

```text
142.250.196.46
```

它根本不认识：

```text
www.google.com
```

所以：

电脑第一句话就是：

> **DNS，我问一下 Google 的 IP 是多少？**

于是：

```
电脑
 │
 │ "google.com 的IP是多少？"
 ▼
DNS服务器
```

DNS 回复：

```
google.com

↓

142.250.196.46
```

所以：

> **DNS = 电话簿。**

**一句话：**

> **DNS：把域名翻译成 IP。**

---

## 第二步：开始连接 Google

知道 IP 后：

电脑说：

```
我要连接：

142.250.196.46
```

但是：

Google 有很多服务。

例如：

* 网页
* 邮件
* 视频

电脑怎么知道我要网页？

于是：

它会告诉 Google：

> **我要访问 443 端口。**

为什么？

因为：

443 就代表：

> **HTTPS 网页。**

所以：

```
电脑
 │
 │ 142.250.xxx.xxx
 │ Port 443
 ▼
Google
```

---

## 第三步：建立 HTTPS

然后：

浏览器：

和 Google：

开始：

```
HTTPS
```

也就是：

> **SSL/TLS 加密。**

现在：

别人就看不懂通信内容了。

---

## 第四步：Google 回复网页

Google：

把网页发回来。

浏览器：

显示：

```
Google首页
```

你看到网页。

整个过程结束。

---

# 整个流程（以后所有知识点都基于它）

```
输入网址

↓

DNS（查IP）

↓

知道IP

↓

连接443端口

↓

HTTPS加密

↓

网页回来
```

---

# 你现在已经认识了三个东西

| 名词        | 它是什么       |
| --------- | ---------- |
| **DNS**   | 电话簿（域名→IP） |
| **IP**    | 每台电脑的地址    |
| **HTTPS** | 给网页通信加密    |

---

# 第二章：邮件为什么和网页完全不是一回事？

很多人一直以为：

> 邮件 = 网页

其实完全不是。

---

## 先看网页（浏览器）

例如：

你打开

```text
https://www.google.com
```

电脑：

```
浏览器
     │
 HTTPS
     │
Google服务器
```

你点一下网页。

Google立刻回复。

这叫：

> **请求 → 回复**

很简单。

---

# 邮件不是这样！

例如：

你给我发邮件。

你以为：

```
我电脑
   │
   │
你的电脑
```

不是！！

真正是：

```
我电脑

↓

邮件服务器A

↓

互联网

↓

邮件服务器B

↓

你的电脑
```

注意：

**中间有两个邮局！**

所以：

邮件其实和寄快递一样。

---

## 举个最真实的例子

假设：

你在东京。

我要寄东西给你。

现实中不会：

```
我家

↓

你家
```

而是：

```
我家

↓

东京邮局

↓

运输中心

↓

江东区邮局

↓

你家
```

电子邮件也是一样。

---

## 所以邮件有两个动作

第一步：

我要把邮件交给邮局。

第二步：

你去邮局拿邮件。

是不是跟现实一模一样？

所以：

邮件就有两个协议。

---

# SMTP

SMTP：

就是：

> **送邮件。**

一句话：

> **SMTP = 寄邮件。**

例如：

```
我

↓

SMTP

↓

邮件服务器
```

SMTP干的事情：

```
我要寄一封邮件。
```

结束。

---

# POP3

POP3：

就是：

> **收邮件。**

一句话：

> **POP3 = 去邮局拿邮件。**

例如：

```
邮件服务器

↓

POP3

↓

你的电脑
```

---

所以：

整个过程就是：

```
我

↓

SMTP

↓

邮件服务器

↓

POP3

↓

你
```

以后考试看到：

> SMTP

脑子里立刻想到：

📤 **寄**

看到：

> POP3

立刻想到：

📥 **拿**

---

# 那为什么还有 IMAP？

POP3：

特点：

> 拿走。

服务器：

```
📮 邮局

↓

你拿走

↓

邮局没了
```

就像：

快递柜。

拿走就没了。

---

IMAP：

特点：

> 不拿走。

服务器：

```
邮件一直放服务器

↓

电脑看

↓

手机看

↓

平板看
```

所以：

现在 Gmail、

QQ 邮箱、

Outlook

几乎都是：

> **IMAP**

因为：

你手机删了。

电脑也同步删。

---

# 所以记住一句口诀

```
SMTP

Send（发送）
```

发送。

---

```
POP3

Pull（拉下来）
```

收下来。

---

```
IMAP

一直放服务器
```

同步。

---

# 再来看一个你前几天一直困惑的问题

你不是一直问：

> SPF 是什么？

现在终于能讲了。

因为你已经知道：

邮件一定会经过：

```
SMTP
```

SMTP 最大的问题是什么？

它可以撒谎。

例如：

坏人可以说：

```
我是：

ceo@google.com
```

其实：

根本不是 Google。

所以：

SMTP：

最大的缺点就是：

> **可以伪造发件人。**

于是人们发明了：

```
SPF
```

---

## SPF 干什么？

它不看邮件内容。

它只问一句：

> **"你这个IP，真的有资格代表 Google 发邮件吗？"**

举例：

Google 公布了一张白名单：

```
Google允许：

1.1.1.1

2.2.2.2
```

今天：

有封邮件来了。

IP：

```
3.3.3.3
```

收件服务器一看：

```
Google说：

我没让这个IP发邮件。
```

于是：

❌

骗子。

结束。

---

所以一句话：

> **SPF = 检查"发邮件的服务器(IP)"是不是官方允许的。**

不是看内容。

不是看附件。

不是看发件人名字。

而是：

> **看发件服务器的 IP。**

# 第一部分：邮件安全（考试最喜欢）

邮件最大的问题：

> **骗子可以冒充别人。**

于是发明了三兄弟。

---

# 第一兄弟：SPF

只检查：

> **谁发的？**

准确一点：

> **发邮件服务器(IP)是不是官方的？**

例如：

```
骗子服务器(IP:3.3.3.3)

↓

说：

我是Google
```

Google：

```
我的服务器只有

1.1.1.1

2.2.2.2
```

SPF：

```
3.3.3.3？

不是。

滚。
```

一句话：

> **SPF 看 IP。**

---

# 第二兄弟：DKIM

SPF解决不了一个问题。

例如：

Google真的发了。

```
Google

↓

邮件

↓

网络
```

途中：

黑客改了一下内容。

例如：

原来：

```
工资500万
```

改成：

```
工资50万
```

怎么办？

于是：

Google在邮件最后：

签个名字。

```
Google

↓

邮件

↓

数字签名
```

收件人收到以后：

重新算一次。

如果：

```
签名一致
```

说明：

没人改。

如果：

```
签名不同
```

说明：

中途被改。

所以：

DKIM根本不是看IP。

它只看：

> **邮件有没有被改。**

一句话：

> **DKIM 看内容。**

---

## 用快递理解

SPF：

看的是：

```
是不是顺丰快递员送来的？
```

DKIM：

看的是：

```
快递有没有被拆过？
```

是不是一下懂了。

# 第三个：DMARC

很多人看到DMARC就懵。

其实：

它根本不是新的验证。

它只是：

**老板。**

它负责：

```
SPF结果

+

DKIM结果

↓

怎么办？
```

例如：

老板写规定：

```
如果失败

↓

直接拒收
```

或者：

```
失败

↓

放垃圾箱
```

或者：

```
失败

↓

先收

但告诉我。
```

所以：

DMARC不是检查。

DMARC：

只是：

> **制定规则。**

一句话：

> **DMARC 看策略。**

---

所以以后考试：

```
SPF

↓

IP
```

```
DKIM

↓

内容
```

```
DMARC

↓

怎么处理
```

三秒结束。

# DNS 又是什么？

不知道

```

```

在哪。

DNS就是：

电话簿。

例如：

```
Google

↓

142.xx.xx.xx
```

浏览器：

先问DNS。

DNS回答：

```
Google在：

142.xx.xx.xx
```

浏览器：

哦。

去连接。

---

所以：

DNS：

一句话：

> **网址→IP地址。**

# DNSSEC

DNS也会骗人。

例如：

黑客回答：

```
Google？

在：

6.6.6.6
```

其实：

假的。

怎么办？

于是：

DNS服务器：

回答的时候：

顺便签个名。

```
IP

+

数字签名
```

浏览器：

验签。

如果：

签名正确。

说明：

没人改。

所以：

DNSSEC：

一句话：

> **DNS答案有没有被篡改。**

注意！

它：

**不是加密。**

很多人会记错。

它只负责：

真实性。

---

口诀：

```
DNS

查地址
```

```
DNSSEC

查答案真假
```

服务器：

在大阪。

```
ssh root@server
```

瞬间：

控制大阪服务器。

全程加密。

一句话：

# VPN

VPN：

什么意思？

很多人以为：

翻墙。

其实：

不是。

VPN真正意思：

就是：

**拉一条私人专线。**

例如：

你在家。

公司：

在东京。

互联网：

大家都能走。

VPN：

```
家

==================

公司
```

别人：

进不来。

而且：

里面：

全部加密。

---

VPN里面：

考试最喜欢两个。

---

## IPsec VPN

工作：

整个网络都能连。

例如：

整个公司。

```
北京公司

========

东京公司
```

全部互通。

---

## SSL VPN

工作：

给个人。

例如：

你在家。

打开浏览器。

登录：

```
https://vpn.company.com
```

结束。

所以：

SSL VPN：

不用装很多客户端。

浏览器就能用。

---

口诀：

```
IPsec

公司和公司
```

```
SSL VPN

个人回公司
```
