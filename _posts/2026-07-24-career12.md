然后：

黑客开始攻击网页。

于是：

```
浏览器
     │
     ▼
Web服务器
```

考试开始考：

| 攻击                   | 意思             | 防御                       |
| -------------------- | -------------- | ------------------------ |
| SQL Injection        | 往SQL塞代码        | Prepared Statement（Bind） |
| XSS                  | 往网页塞JavaScript | Escape（转义）               |
| CSRF                 | 偷偷替别人提交请求      | Token                    |
| OS Command Injection | 往系统命令塞代码       | 不执行Shell、白名单             |
| Directory Traversal  | ../ 偷文件        | 限制目录                     |

然后：

谁负责挡？

```
浏览器

↓

WAF

↓

Web服务器
```

所以：

> **WAF = 专门保护 Web。**

# 第二章：IP 到底是什么？

很多教材直接说：

> IP地址是网络层地址……

第一次学的人都会懵。

其实它就是：

## 📮 门牌号

例如：

你家：

```
東京都江東区……
```

别人才能找到你。

电脑也是一样。

例如：

```
192.168.1.10
```

就是电脑的地址。

所以：

> **IP = 每台电脑的门牌号。**

---

## 为什么需要 IP？

假设世界上没有地址。

快递员：

```
给王强。
```

王强在哪？

不知道。

所以必须有：

```
中国
↓

东京

↓

江东区

↓

某栋楼

↓

301
```

电脑也是一样。

必须知道：

```
IP
```

才能找到对方。


```
SYN

你好。

我想连接。
```

第二句：

```
SYN + ACK

收到。

我也准备好了。
```

第三句：

```
ACK

好。

开始聊天。
```

结束。

连接建立。

所以：

三次握手。

其实就是：

> **双方互相确认，都能正常收发数据。**

---

# 为什么不是两次？

假设：

只有：

```
我：

喂？
```

```
你：

在。
```

万一：

最后一句：

没到。

双方状态不同。

可能一边觉得连上了，一边没有。

所以：

第三次：

告诉对方：

```
好的。

我真的收到你的回复了。
```

双方才真正开始通信。

打给谁。

决定：

找谁。

```
互联网
```

# 📚 我的计划（我觉得会比市面上的教材更适合你）

如果我们继续这样学，大约 **12～15 节**，你就能把 FE 网络与安全模块彻底串起来。

后面的顺序我建议是：

1. **OSI 七层模型**（终于知道为什么有七层，而不是死背）
2. **交换机（Switch）和路由器（Router）的区别**
3. **MAC 地址到底是什么**

6. **Web 攻击（SQL Injection、XSS、CSRF……）——用黑客视角一次讲懂**
7. **最后做 IPA 真题，你会发现每一道题都能放回这张地图。**

我很有信心，按这个顺序学，你以后看到任何网络安全知识，都不会再觉得是零散的缩写，而是知道**它为什么存在、解决什么问题、在整个互联网的哪个位置**。
